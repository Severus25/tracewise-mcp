## Design Patterns Used

1. **Adjacency List with Bidirectional Indexes** вҖ” Forward + reverse indexes enable traversal in both directions with no full-scan
2. **Strategy Pattern** вҖ” Each MCP tool and REST endpoint is a separate query strategy over the same graph
3. **Builder Pattern** вҖ” Graph construction via sequential `addArtifact` / `addLink` calls; easily extended for production connectors
4. **Confidence Propagation** вҖ” Multiplicative decay across BFS edges (inspired by PageRank); deeper links carry lower confidence
5. **Command Pattern** вҖ” MCP tool calls are serializable JSON-RPC commands; same operations available via HTTP and CLI
6. **Facade Pattern** вҖ” `TraceabilityGraph` is the single facade for all analysis; no client needs to know internal indexes
